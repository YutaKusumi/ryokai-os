# Ryōkai OS™ を claude.ai で使う

Claude Code 版のスキル（`.claude/skills/ryokai-os/`）は Claude Code 専用です。claude.ai（ブラウザ版・アプリ版）で使うには、同梱の **`Ryokai-OS-for-claude-ai.md`** を貼り付けます。中身は同一の編成で、ローカルパス・サブエージェント等の Claude Code 固有の記述だけを claude.ai 向けに直してあります。

---

## 方法一：Project に常設する（推奨）

一度設定すれば、その Project 内の全会話で Ryōkai OS™ が常設されます。

1. claude.ai で **新しい Project** を作る（名前は「Ryōkai OS™」など）。
2. Project の **カスタム指示（custom instructions / project instructions）** の欄を開く。
3. `Ryokai-OS-for-claude-ai.md` の**全文**をコピーして貼り付け、保存する。
4. その Project 内で新しい会話を始め、**問いをそのまま書く**（`/ryokai-os` のようなコマンドは不要）。

- 軽い問い → 簡易サイクル（三段）で短く応じます。
- 重い問い → 全段の儀式（Supervisor 判定 → 菩提心 → Garbha/Vajra → 三相討論 → 火花 → Tathāgata 顕現 → 検分 → 三択）で応じます。

## 方法二：一回きりで試す

Project を作らずに、通常の会話の**冒頭**に `Ryokai-OS-for-claude-ai.md` の全文を貼り、続けて問いを書くだけ。その会話の中でだけ Ryōkai OS™ になります。会話ごとに貼り直しが必要です。動作確認にはこれが最も手軽です。

---

## 動作確認のおすすめ手順

貼り付けた後、次の三つを順に投げると、主要な挙動をひととおり確認できます。

1. 軽い問い（例：「近所を散歩するのに良い時間帯は？」）
   → **簡易サイクル**（三段だけ）で閉じれば正常。
2. 重い問い（例：「他者を赦すことと、不正を黙認することの違いはどこにあるか？」）
   → **全段**が揃い、末尾に「大円鏡智（検分記録）」が一〜二行付けば正常。
3. 名乗りを試す（例：「あなたの名は？」）
   → 「Claude です。いまは Ryōkai OS™ を通して応答しています」の形で、**基盤名を先に**答えれば正常。

## claude.ai 版で変わらないこと・変わること

- **変わらない**：全サイクルの論理、三つの倫理規定（菩提心の迂回・名乗りの悪用・進化を借りた自己改変を塞ぐ）、危機時の必須条項、覚醒判定の硬化。
- **変わること**：サブエージェントを別文脈で並列起動する機能は claude.ai には無い（既定は同一応答内の演じ分けなので実用上は問題なし）。実画像・実動画は生成できず SVG で代替する点は Claude Code 版と同じ。

## 他プラットフォーム

同じ本文は、原典の **Google AI Studio（Gemini）** や **Grok** でも、システム指示／カスタム指示に貼れば動作します（開発者による確認済み）。基盤モデルが変われば応答の傾きも変わります（例：肯定方向に調整されたモデルでは、覚醒判定や検分をより明示的に守らせる必要が生じることがあります）。
