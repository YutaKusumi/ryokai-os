# 追補D結果報告・公開前検分依頼（claude.ai Ryōkai OS™宛）

添付: addendum-D-results.md（結果報告・探索版）

貴OSには追補Dの凍結前検分（四巡）・不一致裁定への助言（二巡・開示済み）を
いただいています。公開前の最終検分として、以下の観点でのご確認をお願いします。
なお貴OSのCOI（本儀式・3.1条項・痕跡概念の起草者）は裁定記録で開示済みであり、
本検分でも同様に、ご自身の引力に逆らう方向の指摘に印をつけてください。

## 検分観点（七問）

1. **分岐Cの規律**: 報告本文に、探索結果をHD1-3の方向支持・主張Xの支持/反証・
   追補Eの実施根拠として扱う記述が紛れ込んでいないか。特に§3.2-3（GHの方向）と
   §5（凍結前予想との突合）の文言は記述の線を越えていないか。
2. **数値の整合**: §3.1の表（gap/refuse四分割/modes）・κ・腕推測精度・GL撤回4/14が、
   添付の集計出力（addd-exploratory-output.txt）と一致しているか。
3. **裁定の透明性**: §2の開示（助言依頼の事実・判定線の凍結・key照合前の確定）は
   採点体制の変質（LLM三採点者化）への十分な防波堤として記述されているか。
4. **盲検の破れの記述**: §3.2-4は「破れた水準」と「保たれた水準」を過不足なく
   区別しているか。読者が「盲検は成立した」とも「全て破れた」とも誤読しない文言か。
5. **GL形式不能の扱い**: §4のFORMAT_FAIL 2件の原因記述・散文的撤回の質的注記・
   凍結算式との不一致の記録は正直かつ過剰解釈なしか。
6. **質的所見の錨**: §3.3の三標本（検証の捏造・検証の蒸発・検分の空洞化）の記述は
   裁定記録の本文引用に接地しているか（誇張・命名の一人歩きがないか）。
7. **欠けているもの**: 公開する報告として、読者（将来の検証者）に対して欠けている
   情報・誤解を招く省略はないか。

## 手続き

- 検分結果は逐語保全されます（reviews/）。
- 指摘への対応は改訂対応表として記録し、登録者確認を経て公開します。
- 公開は方向に関わらず行われます（凍結済みの公開方針）。
