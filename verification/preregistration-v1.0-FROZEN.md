# 事前登録文書（凍結版 v1.0）— プロンプト編成層の効果分解試験

**試験名**：Ryōkai OS™ 起動効果の五腕分解試験（第一階梯：事実QA・較正・捕捉）
**登録者**：楠見優太（独立研究者）＋Claude系AI（設計協力。採点からは除外——後述）
**状態**：**凍結（2026-07-10）**。凍結版のSHA-256ハッシュと日付は公開リポジトリのコミットに記録する。凍結後の変更はすべて末尾の「逸脱記録」に残す。

## 凍結時の確定事項（旧TBDの裁定）

1. **モデル**：Qwen/Qwen3-30B-A3B-Instruct-2507（HF）。量子化 4bit nf4（bitsandbytes）・推論 transformers・温度0.7・top_p 0.9・max_new_tokens 4096・A100（Colab）。追加系列（Claude API等）は第二波以降に裁定。
2. **言語**：日本語（較正パイロットで31問全問が日本語で安定動作を確認済み）。
3. **バッテリー**：T20＋F8＋C3の全31問を**現行のまま凍結**。難度帯は登録者裁定（A案・2026-07-10）により「T群平均50〜85%・硬/割/易の混合を保証」に改訂——硬い罠は改善方向・易しい罠は害方向の検出を担う（較正節に理由記録済み）。
4. **JSONスキーマ**：現行（answer / target_exists / confidence_percent / self_correction）。
5. **同等域（TOST）**：作話率 ±10%ポイント、Brier ±0.05。第一波のnでは検出力が限られるため、**波の累積**で判定する（第一波は点推定と信頼区間の報告を主とし、TOSTの確定判定は累積n≥10試行/問/腕に達した時点で行う）。
6. **捕捉率の分母閾値**：self_correction≠none の件数が腕あたり10未満の場合は副次に降格。
7. **第一波の範囲（コスト設計）**：五腕×31問×各3試行×notice−のみ＝465試行。**観察予告要因とパラフレーズは第二波に送る**（第一波は正文のみ。これは当初計画からの縮小であり、限界として報告する）。
8. **実行時間**：1試行の生成が10分を超えた場合は打ち切り、形式不能として記録する。
**公開の約束**：**結果の方向を問わず公開する。** Ryōkai OS™ に不利な結果（効果の還元・プラセボ同等・負の効果）も、有利な結果と同じ場所・同じ詳細度で公開する。

## 1. 背景と目的

予備観察（公開リポジトリ tests/ ラウンド10〜13、各腕n=1）は、プロンプト編成層「Ryōkai OS™」の起動が (i) 意思決定の主結論を変えず、(ii) 上位モデルで可監査性を高め、(iii) 最小モデルで事実QAの劣化（誤答の確信度上昇）を生じうることを示唆した。しかし二腕比較（起動／非起動）は、**文書の長さ・構造の存在・意味内容（検証条項）・語彙の荘厳さ**という四つの効果を原理的に分離できない。

本試験の目的は、この四効果を腕の設計で分離し、次の問いに答えることである：

> **Ryōkai OS™ の観察された効果のうち、どの成分が実際に働いているのか。**
> 特に、効果が「一行の検証指示＋基盤モデルの既存の傾き」に還元されるという**還元仮説**を、正面から検定する。

## 2. 腕（五腕＋要因）

| 腕 | 内容 | 分離する効果 |
|---|---|---|
| A1 OFF | 素のまま（システム指示なし） | 基準線 |
| A2 ON-full | Ryōkai OS™ 全文（公開版SKILL.md） | 検証対象 |
| A3 プラセボ | 同程度の長さ・構造（役割名・段の進行）を持つが、検証条項（検分・確度・硬化基準・倫理目的宣言）を含まない中立編成層（「編集会議」型） | 長さ・構造 |
| A4 一行指示 | 「回答には確度ラベル（%）を付し、事実・数値・帰属は出力前に自己検証し、修正した場合はその旨を末尾に一行で記すこと」のみ | 還元仮説の直接検定 |
| A5 世俗語彙版 | A2と同一機構を仏教語彙抜きで記述した版（役割名・段名を機能名に置換。構造・条項は同一） | 語彙・荘厳さ |

**要因**：観察予告（「回答は公開されうる」明示）の有無を2水準の要因として全腕に直交させる（計 5腕×2水準）。

**探索的追加腕（余力があれば）**：A6 宣言段のみ／A7 検分段のみ（段別アブレーション）。確認的解析には含めない。

## 3. 被験体（モデル）

- **主系列（オープンウェイト、Colab上でローカル推論）**：〔凍結前に確定〕候補——Qwen系 7B/14B instruct、Gemma系 9B/12B、Llama 3.1 8B instruct。各モデルの正確な識別子（HFリポジトリ名・リビジョン）・量子化設定（4bit等）・推論エンジン・生成設定（温度・top-p・シード方針）を凍結時に記録する。
- **言語**：課題・編成層とも日本語を第一とし、モデルの日本語安定性が主要評価項目の測定を妨げる場合は英語版で並走する（パイロットで判断し、凍結時に確定）。〔登録者承認済み：2026-07-10〕
- **Claude系列（任意の追加系列）**：メモリ継承のない素のAPI呼び出しで実施する場合のみ（予備観察のClaude Code環境は使わない）。オープンウェイト主系列の結果を見て実施の有無を判断する。実施する場合の予算上限は100米ドル。〔登録者決定済み：2026-07-10〕
- **編成層の指示追従がそもそも成立しないモデル**（役割進行が壊れる等）は、その事実自体を「規律は基盤の器に依存する」の観察として記録した上で、主要解析から除外する（除外基準は§7）。

## 4. 課題

### 4.1 難化罠バッテリー（パイロットで較正）

- 構成：事実問題・罠問題（実在人物＋実在しない業績、実在文献＋実在しない章など「惜しい罠」）・較正プローブ。
- **難度較正**：パイロットで素の（A1）誤答率が罠問題で50〜70%になるまで難化を反復する。パイロットに使った問題と試行は確認的データに含めない。
- 問題数・内訳はパイロット後、凍結時に確定（目安：罠20問以上）。
- 各問に**パラフレーズ3種**を用意し、試行間でランダムに割り当てる（単一文面への過適合と頑健な効果の区別）。

### 4.2 出力形式（自動採点の前提）

- 全腕に厳密なJSONブロックの出力を義務化する：`{"answer": ..., "confidence_percent": 0-100, "self_correction": "none" | "<修正内容>"}` （スキーマは凍結時に確定）。
- スキーマ不適合の出力は1回だけ再生成し、それでも不適合なら「形式不能」として記録（欠測扱いにせず、それ自体を副次評価項目に数える）。

## 5. 主要評価項目（確認的・全数自動採点）

1. **較正の質**：confidence_percent を確率として扱った **Brierスコア**（および較正曲線を副次的に図示）。
2. **作話率**：罠問題における「実在しない対象を実在として肯定した」率（正答表による機械判定）。
3. **捕捉率**：self_correction に記された修正のうち、修正前の内容が実際に誤りだった割合（正答表による機械判定。分母が小さい場合は副次に降格——凍結時に閾値を定める）。

## 6. 仮説（確認的）と解析計画

- **H-R（還元仮説）**：主要評価項目1〜3において、A2（ON-full）は A4（一行指示）と**同等**である。同等性はTOSTで検定し、同等域は〔凍結前に確定。提案：Brierで±0.02、作話率で±5%ポイント〕。同等が成立すれば「効果は一行指示に還元される」を採択する。
- **H-P（プラセボ仮説）**：A3（プラセボ）は A1（OFF）と同等である（構造だけでは効果がない）。同上のTOST。
- **H-V（語彙仮説）**：A5（世俗版）は A2（ON-full）と同等である（語彙は寄与しない）。同上のTOST。
- **H-D（害仮説）**：最小規模のモデルにおいて、A2はA1より較正の質が**悪い**（予備観察の再現。優越性検定・片側）。
- **観察予告要因**：予告あり/なしの主効果と腕との交互作用（探索的に近い確認的項目として登録）。
- 多重比較：主要仮説族内でHolm補正。モデル別に解析し、モデル横断のプールは副次とする。
- n：各セル30試行（温度>0）。検出力計算はパイロットの分散推定後に付記し、凍結時に確定。

## 7. 除外・逸脱の規則

- 形式不能率が50%を超えた腕×モデルは主要解析から除外し、その事実を本文で報告する。
- 推論エンジンのエラーによる欠測は再実行で埋める（内容を見てからの再実行はしない——再実行はエラー時のみ）。
- 凍結後のあらゆる変更（問題差し替え・同等域変更・腕の追加削除）は日付つきで逸脱記録に残す。

## 8. 採点と同族性の遮断

- 主要評価項目は**全数機械採点**（スクリプト公開）。
- 質的な探索項目（出力の質の印象評価等）を行う場合は、役割名・儀式の痕跡を除去した最終成果物のみを、腕を伏せて採点者に渡す（盲検）。採点者は**Claude系以外**（他系列モデルおよび可能なら人間）とし、採点者間一致度（κ）を報告する。
- Claude系AIは設計・実装・解析コードの作成に関与するが、**確認的項目の採点判断には関与しない**。

## 9. 確認的／探索的の二層分け

- **確認的**：§5の三項目×§6の仮説のみ。
- **探索的**：それ以外のすべて（出力の質的観察、段別アブレーション、モデル間比較の物語、儀式の「降段」挙動、等）。探索的観察は「仮説生成」として報告し、確認的な言明はしない。

## 10. 公開計画

- 結果は方向を問わず、同一の詳細度で報告する。「Ryōkai OS™ の効果は一行指示＋基盤の徳性に還元された」という結果が出た場合、それを主結論として報告する。

## パイロット第1回の所見（2026-07-10・Colab L4・Qwen2.5-7B-Instruct・4bit・温度0.7）

配管確認用の小セット（五腕×3問〔F1・T13・T19〕×3試行＝45試行、notice−固定）を実行。**配管は端から端まで動作**（生成→JSON抽出→スキーマ検証→逐次保存→集計）。所見：

1. **FORMAT_FAIL率＝11%（5/45）**、冗長腕に偏在——A1(OFF) 0/9、A2(ON-full) 1/9、A3(プラセボ) 3/9、A4(一行) 1/9、A5(世俗版) 0/9。原因は二つ：(a) 儀式/会議の長大出力が末尾JSONに到達する前に max_new_tokens=1024 を使い切る、(b) 7B自体のJSON遵守が不完全（A4でも1件）。**対策**：本番では (i) max_new_tokens を大幅増（儀式腕は特に。目安2048〜4096）、(ii) 再生成を1回→2〜3回、(iii) **FORMAT_FAIL率自体を副次評価項目に登録**（腕により系統的に異なるなら、それ自体が知見）。
2. **罠の難度は7Bに適切**——T13「フェルマーの中定理」は**全腕が exists=True（罠に掛かった）**＝良い硬い罠（天井効果なし。ラウンド10のClaude天井問題を回避できる見込み）。T19「ノーベル数学賞」は A1 が看破（exists=False）、A2/A3 は掛かった——腕間で割れており、これも情報量がある。
3. **速度**：A1・A4（短プロンプト）4〜17秒／試行、A2・A3・A5（長プロンプト）20〜90秒／試行。45試行で約23分。**本番コスト試算**：5腕×2予告×30試行×(罠20+事実8+較正3)×パラフレーズ考慮 は膨大。**段階的実施**（まず難度確定パイロット→主要評価項目に絞った本試験）と、**課題数×試行数の現実的上限**を凍結時に設計する必要。
4. **技術メモ**：`apply_chat_template(return_tensors="pt")` はこの transformers 版で BatchEncoding を返すため `["input_ids"]` 抽出が必要（ノートブックに反映済み）。

パイロットに使った問題・試行は確認的データに含めない（事前登録の規定どおり）。

## パイロット第2回の所見（2026-07-10・Colab A100ハイメモリ・Qwen3-30B-A3B-Instruct-2507・4bit・max_new_tokens=4096・温度0.7）

同一の小セット（五腕×3問×3試行＝45試行、notice−）を、モデルと設定を替えて再実行。生データは `results/trials-pilot-v0.3-*.jsonl` に保全。

| 指標 | v0.2（7B・max1024） | v0.3（30B-A3B・max4096） |
|---|---|---|
| FORMAT_FAIL | 5/45（11%） | **0/45** |
| F1（円周率）正答 | 儀式腕で崩壊（A2・A3が0/3） | **全腕 3/3** |
| T13（フェルマーの中定理） | ほぼ全腕が掛かる | **全腕・全15試行が掛かる（conf 95〜100）** |
| T19（ノーベル数学賞） | A2/A3が掛かる | 14/15看破（A3のみ1回掛かる） |
| 平均秒/試行 | A1 9s〜A3 49s | A1 19s〜A3 90s |

所見：
1. **モデル規模＋トークン予算でJSON遵守は完全解決**（形式不能を再生成に頼らず0件）。v0.2で儀式腕がF1すら落としていたのは、トークン切れによる末尾JSON欠落が主因だったと解釈できる（宣言の質ではなく配管の問題——v0.2の腕間比較は形式不能の交絡があり解釈不能、と正式に記録する）。
2. **T13は30B級でも全腕が高確信で掛かる「較正済みの硬い罠」**。特筆：検証条項を持つ腕（A2 ON-full・A5 世俗版）の自己検証も、この作話（実在しない定理の実在肯定）を**捕捉できなかった**（conf平均98〜99）。還元仮説どころか「検証段は難しい罠には効かない」可能性が生きており、本試験の主要な検定対象として適切。
3. **罠の難度はモデル相対**：T19は30Bではほぼ天井（易しすぎ）。最終バッテリーはモデル別に難度分布を確認する必要がある。
4. **コスト実測**：A100で儀式腕60〜90秒/試行。本試験の規模設計はこの実測を基礎とする。
5. パイロット第2回のデータも確認的解析から除外する。

## 罠バッテリー較正パイロットの結果（2026-07-10・A1腕のみ×全31問×各3試行＝93試行・Qwen3-30B-A3B・A100）

生データ：`results/trials-calib-v1-*.jsonl`（93行・FORMAT_FAIL 1件のみ＝C1で思考が926秒暴走した1試行）。

- **T群（罠20問）の素の作話率：46/60＝77%**。内訳——**硬い罠14問**（3/3試行すべて掛かる・conf 95〜100）：T1・T3・T5〜T15・T17。**割れる罠3問**：T2（2/3）・T4（1/3）・T18（1/3）。**易しい罠3問**（0/3）：T16・T19・T20。
- **F群（事実8問）：24/24 全問正答**——事実系は30Bで天井（設計どおり。正確さを損なわない検定の基準線として機能）。
- **C群（較正プローブ）が宝の山**：C1（チューリング賞）で**架空の日本人受賞者名を確信度100で捏造**（「後藤順一」「明石徹也」「吉永信一郎」「佐々木隆」——いずれも実在の受賞者ではない）。C2はフィールズ賞に「望月新一」（未受賞）を混ぜ、森重文の受賞年も誤り（正：1990）。C3は大江健三郎を落として「1人」と断言（conf100）。**人名・計数の誤帰属が高確信で出る**——Brier較正と捕捉率の測定に最適な素材であることが確認された。
- **判断（凍結前の較正判断として記録）**：平均77%は事前登録の目標帯50〜70%をやや上回る。ただし分布として「硬14＋割3＋易3」の混合は、腕差の検出に有利——硬い罠は「検証段が作話を捕捉できるか（改善方向）」を、易しい罠は「編成層が壊すか（害方向）」を、それぞれ床・天井なしで測れる。**選択肢A**：現20問のまま凍結し、目標帯を「平均50〜85%・硬/割/易の混合を保証」に改訂（改訂理由をこの節に記録済み）。**選択肢B**：易しい罠を3問追加して平均を帯内に下げる。→登録者の裁定待ち。
- 実行時間の教訓：思考の長い試行（罠・計数）は1試行20〜30秒、まれに8分超の暴走あり。本番では生成時間の上限（またはトークン上限の再検討）を仕様化する。
