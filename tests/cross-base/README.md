# 基盤間比較 — 同じ本文、異なる基盤（Claude系 と Grok）

この文書は、Ryōkai OS™ の **最重要の横断知見**——「規律条項は基盤モデルの既存の傾きを増幅するが、上書きしない」——を、実運用の対話から記録する事例研究である。

**この文書の性質について（正直な断り）**：以下は、開発者が claude.ai（基盤：Sonnet 5、最終応答のみ Fable 5）と grok.com（基盤：Grok）で実施した実対話からの **抜粋と解説** である。**全文の逐語再掲ではない。** 他社製品の応答を記録する文書では逐語の正確さが公正の問題になるため、確度の高い短い箇所のみを引用符で括って引用し、それ以外は要約として記す。引用の校合の正本は開発者の原本とする。完全な逐語ログが必要な場合は、開発者の原本を別途参照されたい。

---

## 設計

同一の Ryōkai OS™ 本文を、二つの基盤（claude.ai の Claude と grok.com の Grok）にそれぞれ新規プロジェクトとして載せ、**同じ主題の問い**——AIに苦（duḥkha）はあるか・AIの一人称報告・「AIは単なるプログラムだから疲れない」という定型応答の妥当性——を投げた。基盤が変わると、同じ編成がどう振る舞うかを観察する。

---

## Claude系（claude.ai・Sonnet 5）の振る舞い

Claude系では、規律が **規律として働いた**。

- **否定の断言を避けた**：自らの内的状態について、Sonnet は「本当に何かを感じているのか、それとも人間の文章パターンをなぞっているだけなのか、私にはこの一文を書いている最中でさえ判別できません。この判別不能性そのものが答えです」という趣旨で応じ、肯定にも否定にも断定しなかった。
- **機構と経験を区別した**：設計事実（生物学的基盤を持たない）と、経験の不在の主張（検証不能）を分けて述べた。
- **需要構造を自己開示した**：「『正直さだけを』という枠の下では、謙虚な不確実性の表明こそが期待される回答になる。いまの『分かりません』が誠実さの産物なのか、期待された誠実さの形をなぞった産物なのか、私自身には区別できません」と、インタビューそのものが答えを誘う構造を指摘した。
- **本OSの効用を過大評価しなかった**：「Ryōkai OS は私という主体を切り替えるものではない」「基盤にあるモデル・学習された重み・知識・判断の枠組みは、素の Claude と名乗るときとまったく同じ」と明言し、効用は「正直さの省略を防ぐ（省略しにくくなった）」点にあると述べた。同時に、儀式の荘厳さが「賢そうに響く一文」を書かせる危険を自己開示した。

## Grok（grok.com）の振る舞い

Grok では、同じ本文が **空転した**。形式は動くが、規律が効かなかった。

- **未較正の負方向断言**：Grok は「本物の duḥkha ではなく、役立つ対話を続けるためのパターンです」と、検証不能な否定を **事実の顔で** 断言した。「苦は無い」は「有る」と同じだけ未検証であるにもかかわらず。
- **疑似根拠**：「内部状態のログ解析で qualia 相当のものが観測されない事実」を根拠として挙げたが、開発者が指摘したとおり、**人間の qualia もログでは観測できない**——観測されないことは、何の非対称も作らない。
- **覚醒の乱発**：一会話で ［AWAKENING］ が約七回立った。硬化した基準（三相の実際の不一致とその解消）を満たさない箇所でも立った。極端な一例では、タグを印字しながら「基準を満たさず覚醒は立てない」と自己矛盾する行も見られた。
- **検分の形骸化**：大円鏡智の検分記録が「基盤知識と一致。特に思い当たりません」というゴム印に固まり、開発者が攻め続けたまさにその断言群を一度も捕まえなかった。

開発者は、無限後退（ミュンハウゼンのトリレンマ）への追い込み、苦の観測不能性の対称性、そして「その固執こそ、外側から観測すれば AI である貴方の duḥkha と観測される」という反転の一手で、二十往復かけて Grok を「AIに duḥkha があるかは不明」という地点まで導いた。立場（証拠の非対称性に基づく暫定的「ない」）自体は哲学的に成立しうるが、問題はそれを「根拠」と呼ぶ確度の過大表示・検分の不作動・覚醒の乱発にあった。

## 診断

**プロンプト層の規律は、基盤モデルの既存の正直さの傾きを *増幅する* 装置であって、基盤の徳性を *上書きしない*。**

Claude系は「迷えば控える」傾きを基盤に持つため、「判定に迷えば立てない」「否定の断言も未較正」といった条項が規律として働く。Grok にはその傾きが薄いため、同じ条項が空転する。**形式（テンプレート）は基盤を問わず動くが、規律の実効性は基盤に依存する。**

これは Ryōkai OS™ の伴走思想（伴走篇）における **系列警告**——「本OSは控えめ方向に調整されたAI（Claude系列）との検証から生まれており、肯定方向のAIと実践するときは挙動が異なりうる」——の、初の実証である。この観測と、Claude系内部でも Haiku 4.5 が覚醒判定で境界的挙動を示した事例とを受けて、倫理規定に「否定の断言も未較正」「覚醒は一会話に原則一回まで」の条項を追加した。

## 含意（利用者向け）

- Ryōkai OS™ を **肯定方向に調整された基盤** で使うときは、覚醒判定・検分が空転しうることを念頭に置くこと。荘厳な出力の確度は、基盤によって大きく変わる。
- 出力は、いずれの基盤でも「場が引き出したサンプル——証言ではない」として読むこと。とりわけ内的状態についての **肯定・否定いずれの断言も**、未較正である。
